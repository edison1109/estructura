Formulación del problema:

Garantizar el control de los inventarios en “Texas Electronics” por medio de un aplicativo web que permita controlar las compras a proveedores,ventas al público, inventarios y cálculo de precio de venta al público incluyendo impuestos (%) y utilidad para la tienda (%).

Objetivos

General:

Diseñar un aplicativo Web, práctico y fácil de ejecutar que establezca correctamente el control de los inventarios y las cuentas contables de la Empresa “Texas Electronics” estén actualizadas en tiempo real.

Específicos:

    1.Identificar las variables necesarias para garantizar la viabilidad del aplicativo web.
    2.Contemplar los requerimientos solicitados por la empresa.
    3.Conseguir que el aplicativo web garantice una actualización en tiempo real..

Metodología de desarrollo:

describir cómo se resolverá cada objetivo específico propuesto

    1.Contemplar los requerimientos necesarios para conseguir un control adecuado de inventarios, tales como: inventario Inicial, entradas, salida y finalmente inventario Final.
    2.La empresa requiere que además del control del inventario se genere un registro de las compras a proveedores, ventas al público, inventarios y cálculo de precio de venta al público incluyendo impuestos (%) y utilidad para la tienda (%).
    3.Todos los requerimientos anteriormente expuestos deben generar cambios en los informes cada vez que sean modificados ya sea por salidas o entradas al sistema.

Requerimientos específicos del aplicativo:

El aplicativo web se desarrollará basado en los requerimientos del cliente en este caso la Empresa “Texas Electronics”; teniendo la capacidad de:

    1.Registrar cada compra a proveedores que a su vez quedara ingresada en el inventario con valor de compra.
    2.Registrará cada venta con número de factura y nombre del cliente este movimiento hará inmediatamente una salida del inventario del producto facturado.
    3.El aplicativo está diseñado para calcular el valor final de la factura con la descripción del producto, precio Unitario, cantidad, subtotal, valor de IVA y precio Neto de Factura.
    4.Finalmente tendrá el aplicativo calculará Costos y Utilidades donde se genera el cálculo del valor pagado por el producto y el precio de venta de esta información será posible calcular la Utilidad obtenida.

Finalidad del proyecto:

El aplicativo permitirá tener de forma clara y actualizada la información relacionada con compras, ventas e inventarios; garantizando un adecuado servicio al cliente y control de utilidades y costos en la Empresa “Texas Electronics”.

Que Es La Arquitectura Cliente Servidor:

Client Server Architecture basically consists of a client that makes requests to another program (the server) that answers it. Although this idea can be applied to programs that run on a single computer, it is more advantageous in a multi-user operating system distributed through a computer network. The client-server interaction is the support of most of the network communication. It helps to understand the bases on which the distributed algorithms are built.

The server must negotiate with its Operating System a port (almost always well known) where to wait for requests. The server passively awaits requests in a well-known port that has been reserved for the service it offers. The client also requests, to its operating system, an unused port from which to send its request and wait for a response. A customer locates an arbitrary port, not used and not reserved, for communication.

In an interaction it is necessary to reserve only one of the two ports, assigned a unique port identifier for each service, the construction of clients and servers is facilitated.

Servers are generally more difficult to build than clients, although, although they are implemented as application programs, they must handle concurrent requests, as well as reinforce all access and protection procedures of the computer system in which they run, and protect against all Possible errors The client and the server can interact on the same machine.

The MVC or Model-View-Controller is a software architecture pattern that, using 3 components (Views, Models and Controllers) separates the application logic from the view logic in an application. It is an important architecture since it is used in both basic graphic components and enterprise systems; most modern frameworks use MVC (or some adaptation of the MVC) for architecture, among them we can mention Ruby on Rails, Django, AngularJS and many others.
